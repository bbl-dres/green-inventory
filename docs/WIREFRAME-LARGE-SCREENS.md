# Wireframe: Large-Screen Layout Improvements

## Investigation Summary

### Current State Analysis

The Tasks and Dashboard views share a common layout pattern that causes problems on large screens (â‰¥1440px, 1920px, 2560px+):

**Parent container** (`#tasks-view`, `#dashboard-view` in `css/main.css:444-457`):
```css
flex: 1;
display: flex;
flex-direction: column;
align-items: center;      /* Problem: centers children horizontally */
justify-content: center;  /* Problem: centers children vertically */
overflow-y: auto;
padding: 24px;
```

**Inner containers** (`.tasks-view-container`, `.dashboard-container`):
```css
max-width: 1400px;  /* Hard cap â€” wastes space on large screens */
margin: 0 auto;
```

### Identified Problems

| # | Problem | Affected Views | Impact |
|---|---------|---------------|--------|
| 1 | `max-width: 1400px` hard cap | Dashboard + Tasks | On 1920px+ screens, ~500px+ of empty space on each side |
| 2 | `align-items: center` + `justify-content: center` on parent | Both | Content floats in the center when it doesn't fill height; conflicts with scrollable rendered content |
| 3 | No large-screen breakpoints | Both | No `@media (min-width: ...)` rules exist above 1024px â€” only tablet (1024px), mobile (767px), small mobile (479px) |
| 4 | KPI cards stretch too wide | Dashboard | `repeat(auto-fit, minmax(200px, 1fr))` â€” 4 cards in 1400px = 350px each; proportions feel off |
| 5 | Dashboard grid uneven fill | Dashboard | 4 cards in `repeat(auto-fit, minmax(340px, 1fr))` = 3+1 or 4Ã—1 layouts, leaving orphan cards |
| 6 | Calendar month cards too wide | Tasks (Calendar) | `repeat(auto-fill, minmax(280px, 1fr))` â€” 12 months across 4-5 cols, uneven last row |
| 7 | Tasks list table underutilizes width | Tasks (List) | Table is constrained to 1400px; wide screens could show more columns or give breathing room |
| 8 | Yearly planner horizontal scroll on medium screens | Tasks (Planner) | `min-width: 900px` forces scroll at ~900-1400px range, but on large screens it works fine |

## Wireframe Proposals

### Breakpoint Strategy

Add two new large-screen breakpoints:

| Breakpoint | Target | Purpose |
|-----------|--------|---------|
| `â‰¥ 1440px` | Standard desktop / laptop | Widen max-width, refine grids |
| `â‰¥ 1920px` | Full HD / ultrawide | Multi-column layout, utilize full width |
